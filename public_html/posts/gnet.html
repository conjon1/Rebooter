Using Your Phone as a VPN for Your Computer

What Is This?

If you have a phone with a mobile data plan, you can use it as a fully encrypted VPN for your computer — without paying for a separate VPN service for the machine itself.

This setup does three things:

  1. Routes all your computer's traffic through your phone's internet connection
  2. Wraps that traffic in Mullvad VPN running on the phone
  3. Encrypts DNS queries so even domain name lookups are hidden

The end result: your computer's traffic looks like it's coming from a Mullvad server in another country. Your ISP, your network, anyone on the same WiFi — none of them can see what you're doing.

How It Works

The core idea is a chain of tools, each handing traffic to the next:

your app → iptables (intercepts all outbound TCP) → redsocks (converts to SOCKS5) → SSH tunnel (encrypted pipe to your phone) → phone (running Mullvad VPN) → internet

And for DNS (domain name lookups):

your app asks "what is google.com?" → resolv.conf points to 127.0.0.1 → dnscrypt-proxy (encrypts the query over HTTPS) → goes through the SSH tunnel → Mullvad's DNS servers → answer comes back the same way

Without the DNS piece, your computer would leak domain names to your ISP even if all your actual traffic was hidden. dnscrypt-proxy closes that gap.

Startup order (gnet)

Order is strict — each step depends on the previous one being live:

  1. SSH tunnel opens on port 1080
  2. redsocks starts, listens on 12345, forwards to 1080
  3. iptables redirects all outbound TCP to redsocks
  4. dnscrypt-proxy starts, listens on 127.0.0.1:53
  5. resolv.conf set to 127.0.0.1
  6. IPv6 disabled system-wide

Teardown order (gdown)

Reverse of startup — iptables must come down first or network dies. If you kill the SSH tunnel while iptables is still running, all traffic is still being redirected to a process that no longer exists and your network goes completely dead.

  1. iptables rules flushed (restores network reachability immediately)
  2. redsocks stopped
  3. SSH tunnel killed
  4. dnscrypt-proxy stopped
  5. resolv.conf restored from backup
  6. IPv6 re-enabled

Verifying It Works

Once gnet is running, open a browser and go to browserleaks.com/dns or ipleak.net. You should see:

  • IP address: a Mullvad server, not your real IP
  • DNS servers: Mullvad's upstream resolvers (typically Netherlands/UK)
  • IPv6: nothing — IPv6 should be completely absent
  • WebRTC: disable in Brave at brave://flags → search WebRTC → disable

From the terminal:

curl ifconfig.me          # should show Mullvad IP
dig +short google.com     # should resolve (proves DNS works)

What Your ISP Can See

With this setup active, your ISP (T-Mobile or otherwise) sees:

  • That you're sending encrypted data to your phone
  • Nothing else

They cannot see destination websites, DNS queries, content, or metadata. Mullvad can see your destinations but has a strict no-logging policy.

Known Limitations

UDP traffic bypasses the tunnel

redsocks and iptables only handle TCP. Applications that use raw UDP — some games, video calls, and HTTP/3 (QUIC) — will bypass the tunnel entirely. This is a fundamental limitation of SOCKS5 proxying.

NetworkManager may reset DNS on reconnect

If your network interface drops and reconnects, NetworkManager may overwrite /etc/resolv.conf back to the phone gateway, bypassing dnscrypt-proxy. To prevent this permanently:

# /etc/NetworkManager/conf.d/no-dns.conf
[main]
dns=none

gstatus requires sudo mid-run

The iptables check in gstatus needs elevated privileges. To allow it without a password prompt, add this via sudo visudo:

yopi ALL=(ALL) NOPASSWD: /usr/bin/iptables -t nat -L OUTPUT -n

Phone must stay connected

The tunnel depends entirely on the SSH connection to your phone. If the phone locks, USB disconnects, or Mullvad drops, the tunnel dies. Run gdown then gnet to re-establish.

Adapting to Any VPN

The script as written assumes Mullvad on your phone, but the tunnel itself — SSH, redsocks, iptables — works identically regardless of what VPN your phone is running. The only Mullvad-specific part is the DNS server name in dnscrypt-proxy.

Swapping the DNS provider

Change this line in _dnscrypt_config:

server_names = ['mullvad-doh']
providerserver namenotes
Mullvadmullvad-dohdefault in this script
Cloudflarecloudflarefast, US-based, no-log policy
Quad9quad9-doh-ip4-nofilter-ecs-prino-log, malware filtering
AdGuardadguard-dns-dohblocks ads and trackers at DNS
NextDNSnextdnsconfigurable filtering

To list all available servers:

sudo dnscrypt-proxy -config /tmp/dnscrypt-proxy.toml -list

Using no VPN (just routing through carrier)

Skip Mullvad entirely. Your traffic exits through your carrier. Switch DNS to a public resolver:

server_names = ['cloudflare']

Using a self-hosted VPS instead of a phone

Change the config variables to point at your server and update the ssh line in gnet to target your server IP. Everything else stays the same.

GNET_USER="youruser"
GNET_PORT="22"
GNET_PROXY="1080"

Using Shadowsocks instead of SSH

Replace the SSH tunnel section in gnet with ss-local, and update gdown to kill it. redsocks and iptables stay identical — they only care that something speaks SOCKS5 on GNET_PROXY.

ss-local -s YOUR_SERVER_IP -p YOUR_SERVER_PORT \
         -k YOUR_PASSWORD -m aes-256-gcm \
         -l ${GNET_PROXY} -f /tmp/ss-local.pid
The key insight: redsocks and iptables only care that something is listening on GNET_PROXY that speaks SOCKS5. SSH tunnel, Shadowsocks, a local proxy service — any of them drop in without touching the rest of the script.

The Script

Add everything below to your ~/.bashrc. The only values you need to change are the four variables at the very top.

# ─── Phone Tunnel Config ────────────────────────────────────────────────────
GNET_USER="u0_a242"   # Termux username (run `whoami` in Termux to find yours)
GNET_PORT="8022"      # Termux SSH port
GNET_PROXY="1080"     # Local SOCKS5 proxy port
GNET_REDSOCKS="12345" # Redsocks local redirect port

# Helper: returns 0 if tunnel is active, 1 otherwise
_tunnel_active() { ss -tuln | grep -q ":${GNET_PROXY}"; }

# Write redsocks config to a temp file
_redsocks_config() {
    cat <<EOF > /tmp/redsocks.conf
base {
    log_debug = off;
    log_info  = off;
    log       = "syslog:daemon";
    daemon    = on;
    redirector = iptables;
}
redsocks {
    local_ip   = 127.0.0.1;
    local_port = ${GNET_REDSOCKS};
    ip         = 127.0.0.1;
    port       = ${GNET_PROXY};
    type       = socks5;
}
EOF
}

# Write dnscrypt-proxy config — uses Mullvad DoH (HTTPS/TCP, goes through tunnel)
_dnscrypt_config() {
    cat <<EOF > /tmp/dnscrypt-proxy.toml
listen_addresses  = ['127.0.0.1:53']
server_names      = ['mullvad-doh']
max_clients       = 250
ipv4_servers      = true
ipv6_servers      = false
dnscrypt_servers  = false
doh_servers       = true
require_dnssec    = false
require_nolog     = true
require_nofilter  = true
log_level         = 0

[sources.public-resolvers]
  urls = ['https://raw.githubusercontent.com/DNSCrypt/dnscrypt-resolvers/master/v3/public-resolvers.md']
  cache_file = '/tmp/dnscrypt-resolvers.md'
  minisign_key = 'RWQf6LRCGA9i53mlYecO4IzT51TGPpvWucNSCh1CBM0QTaLn73Y7GFO3'
  refresh_delay = 72
EOF
}

# Add iptables rules to redirect all TCP through redsocks
_iptables_up() {
    sudo iptables -t nat -N REDSOCKS 2>/dev/null
    sudo iptables -t nat -A REDSOCKS -d 0.0.0.0/8      -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 10.0.0.0/8     -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 127.0.0.0/8    -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 169.254.0.0/16 -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 172.16.0.0/12  -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 192.168.0.0/16 -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 224.0.0.0/4    -j RETURN
    sudo iptables -t nat -A REDSOCKS -d 240.0.0.0/4    -j RETURN
    sudo iptables -t nat -A REDSOCKS -p tcp -j REDIRECT --to-ports "${GNET_REDSOCKS}"
    sudo iptables -t nat -A OUTPUT   -p tcp -j REDSOCKS
}

# Remove all iptables rules added by gnet
_iptables_down() {
    sudo iptables -t nat -D OUTPUT -p tcp -j REDSOCKS 2>/dev/null
    sudo iptables -t nat -F REDSOCKS 2>/dev/null
    sudo iptables -t nat -X REDSOCKS 2>/dev/null
}

function gnet() {
    if _tunnel_active; then
        echo "tunnel already active on port ${GNET_PROXY}"
    else
        local IFACE PHONE_IP
        read -r _ _ PHONE_IP _ IFACE _ < <(ip route show default | head -n1)

        if [[ -z "$IFACE" ]]; then
            echo "error: no internet connection found -- is USB tethering on?"
            return 1
        fi
        if [[ -z "$PHONE_IP" ]]; then
            echo "error: could not determine phone gateway IP"
            return 1
        fi

        echo "iface    $IFACE"
        echo "phone    $PHONE_IP"
        echo "checking port ${GNET_PORT}..."
        if ! timeout 3 bash -c "echo >/dev/tcp/${PHONE_IP}/${GNET_PORT}" 2>/dev/null; then
            echo "error: port ${GNET_PORT} unreachable on ${PHONE_IP}"
            echo "       is sshd running in Termux? is a VPN blocking it?"
            return 1
        fi

        echo "opening socks5 tunnel on port ${GNET_PROXY}..."
        ssh -f -N -D "${GNET_PROXY}" -p "${GNET_PORT}" \
            -c aes128-gcm@openssh.com \
            -o Compression=no \
            -o IPQoS=throughput \
            -o ConnectTimeout=5 \
            "${GNET_USER}@${PHONE_IP}" || {
            echo "error: ssh failed to start tunnel"
            return 1
        }

        sleep 1
        if ! _tunnel_active; then
            echo "error: tunnel did not open on port ${GNET_PROXY}"
            return 1
        fi
    fi

    echo "starting redsocks..."
    _redsocks_config
    sudo redsocks -c /tmp/redsocks.conf || { echo "error: redsocks failed to start"; return 1; }

    echo "applying iptables rules..."
    _iptables_up

    echo "starting dnscrypt-proxy..."
    _dnscrypt_config
    sudo dnscrypt-proxy -config /tmp/dnscrypt-proxy.toml \
        -pidfile /tmp/dnscrypt-proxy.pid \
        > /tmp/dnscrypt-proxy.log 2>&1 &
    sleep 2
    if ! pgrep -x dnscrypt-proxy > /dev/null; then
        echo "error: dnscrypt-proxy failed to start (check /tmp/dnscrypt-proxy.log)"
        return 1
    fi

    sudo cp /etc/resolv.conf /tmp/resolv.conf.bak
    echo "nameserver 127.0.0.1" | sudo tee /etc/resolv.conf > /dev/null

    echo "disabling ipv6..."
    sudo sysctl -w net.ipv6.conf.all.disable_ipv6=1     > /dev/null
    sudo sysctl -w net.ipv6.conf.default.disable_ipv6=1 > /dev/null

    echo "system-wide tunnel active -- DNS encrypted via DoH"
}

function gdown() {
    echo "tearing down..."
    _iptables_down
    echo "iptables rules removed"

    if pgrep -x redsocks > /dev/null; then
        sudo pkill redsocks && echo "redsocks stopped"
    fi

    local TUNNEL_PID
    TUNNEL_PID=$(ss -lptn "sport = :${GNET_PROXY}" \
        | awk -F'pid=' 'NR>1 { split($2, a, ","); print a[1]; exit }')
    if [[ -n "$TUNNEL_PID" ]]; then
        kill "$TUNNEL_PID" \
            && echo "tunnel stopped (pid $TUNNEL_PID)" \
            || echo "error: could not kill tunnel pid $TUNNEL_PID"
    else
        echo "no active tunnel on port ${GNET_PROXY}"
    fi

    if pgrep -x dnscrypt-proxy > /dev/null; then
        sudo pkill dnscrypt-proxy && echo "dnscrypt-proxy stopped"
    fi
    if [[ -f /tmp/resolv.conf.bak ]]; then
        sudo cp /tmp/resolv.conf.bak /etc/resolv.conf
        sudo rm /tmp/resolv.conf.bak
        echo "dns restored"
    fi

    sudo sysctl -w net.ipv6.conf.all.disable_ipv6=0     > /dev/null
    sudo sysctl -w net.ipv6.conf.default.disable_ipv6=0 > /dev/null
    echo "ipv6 restored"

    if pgrep -x "brave" > /dev/null; then
        pkill brave && echo "brave closed"
    fi
}

function gstatus() {
    local IFACE PHONE_IP TUNNEL_PID PROXY_IP REAL_IP DNS_LEAK

    IFACE=$(ip route show default | awk 'NR==1 {print $5}')
    PHONE_IP=$(ip route show default | awk 'NR==1 {print $3}')

    echo "[ ssh tunnel ]"
    if _tunnel_active; then
        TUNNEL_PID=$(ss -lptn "sport = :${GNET_PROXY}" \
            | awk -F'pid=' 'NR>1 { split($2, a, ","); print a[1]; exit }')
        echo "  status   active"
        echo "  port     ${GNET_PROXY}"
        echo "  pid      ${TUNNEL_PID:-unknown}"
        echo "  phone    ${PHONE_IP:-unknown}  (via ${IFACE:-unknown})"
    else
        echo "  status   inactive"
    fi

    echo "[ redsocks ]"
    if pgrep -x redsocks > /dev/null; then
        echo "  status   active"
        echo "  port     ${GNET_REDSOCKS}"
        if sudo iptables -t nat -L OUTPUT -n | grep -q REDSOCKS; then
            echo "  iptables routing -> active"
        else
            echo "  iptables routing -> missing (run gnet to fix)"
        fi
    else
        echo "  status   inactive"
    fi

    echo "[ ip ]"
    PROXY_IP=$(curl -s --max-time 5 --socks5-hostname "127.0.0.1:${GNET_PROXY}" ifconfig.me 2>/dev/null)
    echo "  tunnel   ${PROXY_IP:-unavailable}"
    if pgrep -x redsocks > /dev/null; then
        if sudo iptables -t nat -L OUTPUT -n 2>/dev/null | grep -q REDSOCKS; then
            echo "  real     (redirected -- redsocks active)"
            echo "  masked   yes (redsocks + iptables confirmed)"
        else
            echo "  masked   no -- redsocks running but iptables chain missing"
        fi
    else
        REAL_IP=$(curl -s --max-time 5 --interface "${IFACE}" ifconfig.me 2>/dev/null)
        echo "  real     ${REAL_IP:-unavailable}"
        if [[ -n "$PROXY_IP" && -n "$REAL_IP" && "$PROXY_IP" != "$REAL_IP" ]]; then
            echo "  masked   yes"
        elif [[ -z "$PROXY_IP" ]]; then
            echo "  masked   unknown (tunnel unreachable)"
        else
            echo "  masked   no -- traffic may not be routing through tunnel"
        fi
    fi

    echo "[ dns ]"
    if pgrep -x dnscrypt-proxy > /dev/null; then
        DNS_LEAK=$(dig +short +time=3 google.com @127.0.0.1 2>/dev/null | head -1)
        echo "  dnscrypt active"
        echo "  resolves via  127.0.0.1 (dnscrypt → DoH → tunnel)"
        echo "  dig           ${DNS_LEAK:-unavailable}"
        if [[ -n "$DNS_LEAK" ]]; then
            echo "  leak          none -- dnscrypt resolving correctly"
        else
            echo "  leak          unknown -- dig failed, check dnscrypt"
        fi
    else
        echo "  dnscrypt inactive -- DNS may be leaking"
        echo "  server   $(awk '/^nameserver/{print $2; exit}' /etc/resolv.conf)"
    fi

    echo "[ system ]"
    echo "  iface    ${IFACE:-unknown}"
    echo "  ttl      $(sysctl -n net.ipv4.ip_default_ttl)"
}
